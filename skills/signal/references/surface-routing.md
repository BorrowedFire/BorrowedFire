# Surface Routing

Select the smallest useful skill set from the Corey Haines Marketing Skills catalog. Always load the referenced skill file before relying on it. For a full catalog, see `corey-haines-marketing-skills.md`.

## Foundation

| Situation | Skill |
| --- | --- |
| Missing or stale product, audience, positioning, ICP, differentiation, or voice context | `product-marketing` |
| Need customer language, objections, JTBD, or proof before writing | `customer-research` |
| Existing copy needs a final tightening pass | `copy-editing` |
| New persuasive marketing copy for a website or page | `copywriting` |
| The user asks why the message is not landing or wants behavioral framing | `marketing-psychology` |
| User wants ideas for how to market, not just wording | `marketing-ideas` |

## Routing Notes

- If the user only says "run Marketing Skills" against copy, infer the surface from the copy and recent context. If recent context is unavailable, ask what surface it is for.
- If multiple surfaces are likely, provide separate tuned versions. Do not reuse one generic line across App Store, social, email, and in-app surfaces without adaptation.
- For Apple App Store promotional text, remember it is conversion copy, not indexed search metadata. Keep it under 170 characters and make it paste-ready.
- For paid ads, generate multiple variants and validate platform character limits.
- For in-product copy, prioritize trust, clarity, timing, and product truth over acquisition-style punch.
- For social, write for the platform's first-line behavior and audience norms, not for a web hero section.
- For emails, give subject, preview, body, CTA, and sequence context when applicable.
